The Vibe Coding Approach

The dashboard employs "vibe coding" - a synthesis of coding, storytelling, and interface design that follows the natural flow of negotiation practice. This approach integrates technical implementation with narrative-driven user experience.

Unlike traditional development that separates frontend, backend, and logic layers, this dashboard unifies them into a cohesive HTML, CSS, and JavaScript application. This design choice offers several advantages:

Integrated Negotiation Frameworks

Island of Agreements

This framework identifies potential starting points for dialogue by mapping areas of convergence and divergence between parties. It sorts negotiation elements into facts vs. norms and establishes positive points of departure.

Click to enlarge
Iceberg Analysis

Visual tool that reveals underlying interests beneath stated positions. It analyzes visible positions (what), reasoning (how), and the deeper motives and values (why) driving both parties in the negotiation.

Click to enlarge
Influence Pathways

A graphical and narrative representation of power relationships and influence dynamics. It maps stakeholders on a 2D grid to develop quadrant-based engagement strategies and identify key influence channels.

Click to enlarge
Stakeholder Matrix

Maps key stakeholders by their power, legitimacy, and urgency to help prioritize negotiation focus. Provides a structured view of stakeholder positions, priorities, and actions needed.

Click to enlarge
Multiparty Dynamics

Maps complex stakeholder relationships and identifies potential coalition-building opportunities. Helps understand group dynamics and formulate effective engagement strategies across different stakeholder coalitions.

Click to enlarge
Tactical Timeline

Structures negotiation approach through critical timeframes with phase-based planning. Creates a clear roadmap for progressive engagement with key milestones and specific objectives for each stage of the negotiation process.

Click to enlarge
Radical Faction Analysis

Focuses on managing small but influential radical factions that could potentially derail negotiations. Provides characterization, management strategies, and risk assessment matrices for dealing with disruptive elements.

Click to enlarge
Critical Success Factors

Identifies key factors that will determine successful resolution of the crisis. Uses visual metrics to track stakeholder trust, alignment, and negotiation process effectiveness while providing targeted recommendations for improvement.

Technical Implementation

The dashboard was intentionally developed as a single-page web application using modern web technologies, making it accessible in humanitarian settings where technical infrastructure may be limited.

Current Architecture

The system loads JSON files corresponding to specific negotiation cases, parsing and rendering them into visual blocks, maps, and dynamic narratives. All logic and interface elements are transparent and modifiable.

  • Single-page web application architecture
  • JSON-driven data structure
  • Dynamic rendering of negotiation frameworks
  • Interactive visualization components
  • Modular and adaptable design
  • Low technical barrier for deployment

Future AI-Enhanced Capabilities

A future iteration of the dashboard will integrate additional capabilities for handling natural language inputs and automatic data structuring:

  • Backend integration with multiple LLMs (GPT, Claude, Gemini)
  • Process automation with tools like n8n
  • Voice agents (e.g., ElevenLabs) for accessibility
  • Translation models for multilingual support
  • Local deployment with open-source LLMs (e.g., Gemma via Ollama)
  • Enhanced visualization with realtime updates

Technical Documentation

Frontend Implementation

  • HTML/CSS/JavaScript: Core technologies used for all interface components
  • Minimal Dependencies: No external frameworks to ensure simplicity and portability
  • JSON Data Structure: All negotiation data stored in structured JSON format
  • DOM Manipulation: Direct manipulation for maximum control and performance

AI Integration Architecture

  • Prompt Templates: Specialized templates for each negotiation framework
  • JSON Schema: Structured output format for consistent visualization
  • Custom GPT: Dedicated model fine-tuned for negotiation analysis
  • Data Transformation: Pipeline for converting AI responses to dashboard format

Deployment Strategy

  • Static Hosting: Deployed on GitHub Pages for global accessibility
  • Offline Capability: Designed to function in limited-connectivity environments
  • Export Options: PDF and HTML formats for sharing analysis
  • Version Control: Git-based system for tracking dashboard evolution
